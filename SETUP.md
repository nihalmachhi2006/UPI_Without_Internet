# Quick Setup Guide

## Prerequisites

| Requirement | Version | Check |
|-------------|---------|-------|
| Python      | 3.10+   | `python --version` |
| pip         | any     | `pip --version` |

That's it. No Python, no Maven, no database, no Redis.

---

## 1 — Clone / Extract

If you downloaded the zip:
```bash
unzip upi_offline_mesh.zip
cd upi_offline_mesh
```

If cloning from GitHub:
```bash
git clone https://github.com/<your-username>/upi-offline-mesh-python.git
cd upi-offline-mesh-python
```

---

## 2 — Create a virtual environment (recommended)

```bash
# Windows
python -m venv .venv
.venv\Scripts\activate

# macOS / Linux
python3 -m venv .venv
source .venv/bin/activate
```

---

## 3 — Install dependencies

```bash
pip install -r requirements.txt
```

This installs FastAPI, Uvicorn, the `cryptography` library, Pydantic, and pytest.
Total download is roughly **15 MB**.

---

## 4 — Start the server

```bash
python main.py
```

You should see:

```
🔐 Generating RSA-2048 keypair...
🗄️  Initialising in-memory database...
✅ UPI Mesh server started — open http://localhost:8080
INFO:     Uvicorn running on http://0.0.0.0:8080
```

Open **http://localhost:8080** in your browser.

---

## 5 — Run the demo (four steps)

The dashboard has four step buttons at the top:

| Step | Button | What happens |
|------|--------|-------------|
| 1 | **📤 Inject into Mesh** | Fill in sender/receiver/amount and click. The server-side DemoService encrypts the payment and injects it into `phone-alice`. |
| 2 | **🔄 Run Gossip Round** | Click 1–2 times. Each round, every device broadcasts its packets to every neighbour. TTL decrements per hop. |
| 3 | **📡 Bridges Upload** | `phone-bridge` (the only internet-connected device) POSTs all its packets to `/api/bridge/ingest`. Watch the balances change and the ledger update. |
| Reset | **🗑️ Clear Mesh** | Wipes all device packet stores and the idempotency cache. Balances are NOT reset (restart the server for a fresh DB). |

---

## 6 — Run the tests

```bash
pytest tests/ -v
```

Expected output:

```
tests/test_mesh.py::test_encrypt_decrypt_round_trip                              PASSED
tests/test_mesh.py::test_tampered_ciphertext_is_rejected                         PASSED
tests/test_mesh.py::test_single_packet_delivered_by_three_bridges_settles_exactly_once  PASSED
```

The third test is the headline one — three threads deliver the same packet simultaneously and the assertion proves exactly one settles.

---

## 7 — Stop the server

`Ctrl+C` in the terminal.

## Troubleshooting

**`ModuleNotFoundError: No module named 'cryptography'`**
→ Run `pip install -r requirements.txt` again inside your virtual environment.

**`Address already in use`**
→ Something is already on port 8080. Change the port in `main.py`:
```python
uvicorn.run("main:app", host="0.0.0.0", port=9090, reload=False)
```

**`python: command not found`**
→ Try `python3 main.py` instead.

**Tests fail with `RuntimeError: ServerKeyHolder not initialized`**
→ Make sure you're running pytest from the `upi_offline_mesh/` directory.

## API quick reference

| Method | Path | Description |
|--------|------|-------------|
| `GET`  | `/` | Dashboard UI |
| `GET`  | `/api/server-key` | RSA public key (base64) |
| `GET`  | `/api/accounts` | All accounts and balances |
| `GET`  | `/api/transactions` | Last 20 settled transactions |
| `GET`  | `/api/mesh/state` | Packet counts on each virtual device |
| `POST` | `/api/demo/send` | Simulate sender phone — encrypt + inject |
| `POST` | `/api/mesh/gossip` | Run one gossip round |
| `POST` | `/api/mesh/flush` | Bridge nodes upload to backend |
| `POST` | `/api/mesh/reset` | Clear mesh + idempotency cache |
| `POST` | `/api/bridge/ingest` | Production endpoint — real bridges POST here |

Interactive docs: **http://localhost:8080/docs** (Swagger UI, auto-generated by FastAPI)
